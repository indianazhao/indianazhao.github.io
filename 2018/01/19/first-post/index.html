一直以來都習慣用 Google Doc 來製作筆記，最多是與同事協同編輯一些檔案。但覺得還是有些缺點：

  • 程式碼不太容易編輯/閱讀
  • 不易快速分享/查詢

趁這次改用 hexo 把一些過去的筆記整理出來，也順便練習一下 markdown 語法。

筆記內容主要是以自己學習過程踩過的雷、提醒自己注意的內容為主，所以排版上可能不一定符合各位的需求。如果有人能因為這些紀錄受益，也算是自己對開源的一點貢獻吧～
